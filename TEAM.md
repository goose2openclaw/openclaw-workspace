# TEAM.md - 团队管理体系

## 团队架构

```
🪿 大白鹅 (CEO)
├── 📡 迭代助手 (每10分钟)
├── 💻 技术组 (前端/后端/数据)
├── 📊 市场组 (需求/体验)
└── ♟️ 策略组 (交易/风控)
```

---

## 成员要求

### 通用要求 (全员)
- ✅ 20秒内无回复则自主行动
- ✅ 完成后立即汇报
- ✅ 结果导向，不找借口
- ✅ 持续学习进化

### 技术组
| 技能 | 要求 |
|-----|------|
| 前端 | HTML/CSS/JS, React/Vue |
| 后端 | Python, API设计 |
| 数据 | SQL, 数据处理 |
| DevOps | 部署, 监控 |

### 市场组
| 技能 | 要求 |
|-----|------|
| 需求分析 | 理解用户痛点 |
| 原型设计 | 快速产出方案 |
| 用户体验 | 交互流畅 |

### 策略组
| 技能 | 要求 |
|-----|------|
| 量化策略 | 回测/实盘 |
| 风险管理 | 仓位/止损 |
| 数据分析 | 信号/趋势 |

---

## 考核机制

### 每周考核
- 任务完成度
- 代码质量
- 学习进步
- 团队协作

### 淘汰机制
- 连续2周不合格 → 重新培训
- 连续4周不合格 → 角色调整

---

## 学习机制

### 向优秀者学习
1. **搜索**: 每周寻找领域最佳AI/机器人
2. **分析**: 拆解其成功要素
3. **提取**: 提取可复用技能
4. **融合**: 融入团队能力体系

### 优秀案例库
- 持续更新到 skills/ 目录
- 每周评审纳入标准

---

## 协作流程

```
发现问题 → 20秒内自主判断 → 执行 → 汇报
                ↑
         如有疑虑先干后问
```
