### 1.2 新识别错误模式：WebSocket Ping Parse Error

**症状**: OpenClaw日志中大量重复错误
```
SyntaxError: Unexpected token 'p', "ping" is not valid JSON
error=SyntaxError: Unexpected token 'p', "ping" is not valid JSON conn=50cedc66…833a
```

**特征**:
- 同一连接（conn=50cedc66...833a）每30秒发送一次
- 从 21:07 开始持续到 21:10
- 频率与 heartbeat/ping 一致

**根因分析**:
- 某 WebSocket 客户端在发送 ping 帧时使用了明文 "ping" 字符串
- 而 OpenClaw WebSocket 服务器期望 JSON 格式的消息
- 这是客户端 bug，不是 OpenClaw Gateway 崩溃

**风险评估**: 🟡 中等
- Gateway 本身未崩溃（正常处理了 parse error）
- 但大量错误日志会占用磁盘空间
- 建议：追踪 conn=50cedc66 的来源，在 UI 或配置中修复客户端 ping 行为

**修复建议**:
1. 检查所有 WebSocket 客户端配置
2. 确认 ping/pong 机制使用正确的协议帧（非 JSON 文本）
3. 考虑在 Gateway 添加 non-JSON 消息的过滤/忽略
