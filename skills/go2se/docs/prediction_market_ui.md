## 三、侦测轨道

### 3.1 数据流

```
API数据获取 → 数据解析 → 模型分析 → 决策判断 → 信号输出
     ↓            ↓           ↓          ↓         ↓
  主/备API    JSON解析    5模型评分   3模式决策   执行/等待
```

### 3.2 侦测流程

```
1. 获取预测市场列表
2. 过滤: 成交量>$100K, 概率55-95%
3. 模型评分: 情绪+成交量+概率趋势+市场情绪+技术
4. 置信度计算
5. 决策: 买入/卖出/等待
```

---

## 四、模型比照

### 4.1 五大模型

| 模型 | 权重 | 功能 |
|------|------|------|
| 情绪分析 | 25% | 社交媒体情绪 |
| 成交量分析 | 20% | 市场活跃度 |
| 概率趋势 | 25% | 概率变化趋势 |
| 市场情绪 | 15% | 整体市场状态 |
| 技术分析 | 15% | K线/形态 |

### 4.2 评分计算

```
置信度 = Σ(模型分数 × 权重)
推荐 = 置信度 > 0.5 ? BUY : WAIT
```

---

## 五、三种模式参数

### 5.1 参数对比

| 参数 | 保守 | 平衡 | 激进 |
|------|------|------|------|
| 置信度阈值 | 0.70 | 0.55 | 0.45 |
| 最大仓位 | 3% | 5% | 8% |
| 止损 | 2% | 3% | 5% |
| 止盈 | 10% | 15% | 25% |
| 最低概率 | 65% | 55% | 50% |

### 5.2 适用场景

- **保守**: 市场不确定性高 / 新手
- **平衡**: 正常市场 / 一般投资者
- **激进**: 高确定性机会 / 高风险偏好

## 六、决策示例

### 6.1 决策流程

```
输入: 市场数据 + 模型分析
   ↓
检查1: 概率 > 最低阈值?
   ↓ 是
检查2: 成交量 > 最小值?
   ↓ 是
检查3: 置信度 > 阈值?
   ↓ 是
输出: BUY/SELL + 仓位/止损/止盈
```

### 6.2 示例输出

```
📊 模式: balanced
🔮 机会: SOL > $500 by 2026?
   概率: 73.7%
   置信度: 58.6%
   决策: BUY
   仓位: 5%
   止损: 3%
   止盈: 15%
```

## 七、极端情况处理

### 7.1 应急预案

| 情况 | 应对 |
|------|------|
| API响应超时 | 切换备用API → 使用缓存 |
| 限流429 | 等待1分钟 → 切换API |
| 拒绝502/503 | 切换备用API → 告警 |
| 数据异常 | 跳过该市场 → 记录日志 |

### 7.2 故障转移

```
主API失败 → 备用1 → 备用2 → 备用3 → 模拟数据 → 告警
```
