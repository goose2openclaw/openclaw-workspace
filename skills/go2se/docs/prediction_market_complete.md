## 三、API管理

### 3.1 API列表

| 优先级 | 名称 | URL | 超时 | 限流 |
|--------|------|-----|------|------|
| 1 | Polymarket | clob.polymarket.com | 15s | 100/min |
| 2 | Polymarket USDT | gamma.polymarket.com | 15s | 100/min |
| 3 | Augur | augur.net | 20s | 50/min |
| 4 | Manifold | manifold.markets | 20s | 50/min |

### 3.2 故障处理

```
超时(15s) → 切换备用API
限流(429) → 等待1分钟 → 切换
拒绝(502/503) → 切换备用 → 告警
全部失败 → 使用模拟数据 → 记录
```

## 四、模型分析

### 4.1 五大模型

| 模型 | 权重 | 输入 | 输出 |
|------|------|------|------|
| 情绪分析 | 25% | 社交媒体 | 0-1分数 |
| 成交量 | 20% | 交易量 | 0-1分数 |
| 概率趋势 | 25% | 历史概率 | 0-1分数 |
| 市场情绪 | 15% | BTC/ETH | 0-1分数 |
| 技术分析 | 15% | K线形态 | 0-1分数 |

### 4.2 置信度计算

```
置信度 = Σ(模型分数 × 权重)
推荐 = 置信度 > 0.5 ? BUY : WAIT
```

## 五、决策逻辑

### 5.1 决策流程

```
市场数据输入
     ↓
检查1: 概率 ≥ 最低阈值?
     ↓ 是
检查2: 成交量 ≥ 最小值?
     ↓ 是
检查3: 置信度 ≥ 模式阈值?
     ↓ 是
输出: BUY + 仓位/止损/止盈
     ↓ 否
输出: WAIT + 原因
```

### 5.2 示例

```
输入:
  - 市场: SOL > $500 by 2026?
  - 概率: 73.7%
  - 成交量: $2,500,000
  - 置信度: 58.6%
  
决策 (平衡模式):
  - 概率 73.7% ≥ 55% ✓
  - 成交量 $2.5M ≥ $100K ✓
  - 置信度 58.6% ≥ 55% ✓
  
输出:
  - 行动: BUY
  - 仓位: 5%
  - 止损: 3%
  - 止盈: 15%
```
