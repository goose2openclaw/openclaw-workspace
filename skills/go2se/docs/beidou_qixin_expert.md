## 二、风险评估系统

### 2.1 风险指标

| 指标 | 计算方法 | 权重 |
|------|----------|------|
| VaR(95%) | 持仓风险敞口 | 30% |
| 组合波动率 | 历史波动 | 20% |
| 集中度 | 持仓分散度 | 25% |
| 市场风险 | BTC波动 | 25% |

### 2.2 风险等级

| 评分 | 等级 | 建议 |
|------|------|------|
| 70+ | 极高 | 立即减仓 |
| 50+ | 高 | 谨慎操作 |
| 30+ | 中 | 正常操作 |
| <30 | 低 | 积极投资 |

### 2.3 警告机制

- ⚠️ 风险极高，建议减仓
- ⚠️ VaR过高，单日可能损失超过10%
- ⚠️ 持仓过于集中

## 三、优化建议系统

### 3.1 优化维度

```
┌─────────────────────────────────────────────────────────────────┐
│                     优化建议                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 模式优化                                                  │
│     • 持仓过多 → 切换保守                                      │
│     • 风险低 → 可以激进                                        │
│                                                                 │
│  2. 工具优化                                                  │
│     • 组合单一 → 分散投资                                      │
│     • 某工具过载 → 均衡分配                                    │
│                                                                 │
│  3. 资源优化                                                  │
│     • 算力不足 → 优先级调度                                    │
│     • API故障 → 备用切换                                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 优化评分

```
分数 = 基础分(50) + 持仓分 + 风险分 + 分散分

- 持仓3-8个: +10
- 风险<50: +20
- 分散>2工具: +10
```

```
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 6.2 学习机制

```python
# 基于历史表现调整参数
performance_history = {
    'rabbit': {'wins': 10, 'losses': 3, 'win_rate': 0.77},
    'mole': {'wins': 5, 'losses': 7, 'win_rate': 0.42},
    'prediction': {'wins': 2, 'losses': 1, 'win_rate': 0.67}
}

# 自动调整权重
for tool, perf in performance_history.items():
    if perf['win_rate'] > 0.6:
        increase_allocation(tool)
    elif perf['win_rate'] < 0.4:
        decrease_allocation(tool)
```

## 七、专家系统参数

### 7.1 核心参数

| 参数 | 范围 | 默认 | 说明 |
|------|------|------|------|
| 市场检测窗口 | 30-100 | 30 | 检测周期 |
| 风险评估周期 | 实时 | 实时 | VaR计算 |
| 优化建议频率 | 1-24h | 1h | 优化周期 |
| 自适应学习率 | 0.01-0.1 | 0.05 | 参数更新 |

### 7.2 行动阈值

| 触发条件 | 行动 |
|----------|------|
| 风险>70 | 强制减仓50% |
| VaR>10% | 告警+建议 |
| 连续3亏 | 暂停工具 |
| 胜率<40% | 降低仓位 |
