## 二、模拟交易系统 (simulated_trading_v1.py)

### 2.1 核心功能

| 模块 | 功能 |
|------|------|
| **声纳趋势模型** | 10+趋势形态识别，准确率65-78% |
| **模拟投资组合** | 仓位管理、止损止盈、开仓平仓 |
| **信号处理** | 信号→趋势匹配→虚拟回报→执行判断 |

### 2.2 工作流程

```
信号输入 → 市场数据获取 → 声纳模型匹配 → 虚拟回报计算 → 执行判断 → 虚拟交易
```

### 2.3 关键参数

| 参数 | 说明 |
|------|------|
| 初始资金 | $10,000 |
| 最大仓位 | 15% |
| 默认止损 | 3% |
| 置信度阈值 | 0.5 |

## 三、真实API对接系统 (real_api_integration.py)

### 3.1 支持API

| API | 优先级 | 限流 | 用途 |
|-----|--------|------|------|
| Binance | 主 | 1200/分钟 | 现货数据 |
| Bybit | 备1 | 600/分钟 | 合约数据 |
| OKX | 备2 | 300/分钟 | 备用 |
| Mock | 测试 | 10000/分钟 | 本地测试 |

### 3.2 核心组件

```
APIManager
    ├── API客户端 (requests)
    ├── 健康检查
    ├── 自动切换
    └── 限流控制

TradingExecutor
    ├── 订单构建
    ├── 订单提交
    └── 状态跟踪

SignalRouter
    ├── 信号验证
    ├── 过滤器
    └── 路由分发
```

### 3.3 故障处理

```
连续3次失败 → 标记为degraded
连续5次失败 → 标记为inactive
自动切换 → 备用API
```

## 四、风控告警系统 (risk_control_system.py)

### 4.1 监控模块

| 模块 | 监控内容 |
|------|----------|
| **PortfolioMonitor** | 仓位限制、日内亏损、止损 |
| **MarketMonitor** | 价格波动、波动率 |
| **APIMonitor** | API健康、错误计数 |
| **ResourceMonitor** | CPU、内存、API配额 |

### 4.2 告警级别

| 级别 | 阈值 | 动作 |
|------|------|------|
| 🔴 CRITICAL | 价格波动>10% | 立即处理 |
| 🟠 HIGH | 价格波动>5% | 快速响应 |
| 🟡 MEDIUM | 波动率高 | 监控 |
| 🟢 LOW | API异常 | 记录 |

### 4.3 自动动作

| 触发条件 | 自动动作 |
|----------|----------|
| API故障 | switch_to_backup |
| 风险过高 | reduce_position |
| 市场剧烈波动 | increase_stop_loss |

## 六、大白鹅调度系统 (goose_scheduler.py)

### 6.1 核心能力

| 能力 | 说明 |
|------|------|
| **深度推理** | 市场状态检测、机会评分 |
| **挑战识别** | 风险/机会/系统/策略/创新 |
| **交互提问** | 向用户请求反馈 |
| **任务执行** | 自动执行+记录 |

### 6.2 调度周期

```
每20分钟运行:
    ↓
1. 深度推理分析 → 市场洞察
    ↓
2. 识别挑战 → 问题分类
    ↓
3. 生成问题 → 向用户提问
    ↓
4. 执行任务 → 记录结果
```
