## 5. 交易工具

### 5.1 开启交易
1. 进入 **设置** 页面
2. 配置交易所 API Key
3. 选择交易模式：
   - **模拟交易** (dry_run) - 不真实下单
   - **实盘交易** (live) - 真实下单

### 5.2 手动交易流程
```
市场选币 → 查看信号 → 确认策略 → 输入金额 → 执行交易
```

### 5.3 策略执行
| 步骤 | 操作 |
|------|------|
| 1 | 在策略页面选择工具 |
| 2 | 配置参数（仓位、止损、止盈）|
| 3 | 点击"执行" |
| 4 | 查看交易记录 |

## 6. 智能系统

### 6.1 MiroFish 仿真引擎

**功能**: 25维度全向仿真评估

**访问**: `/integration/mirofish/run`

**评分标准**:
- 100分：最优
- 70-99分：正常
- 50-69分：警告
- 0-49分：不合格

**25个维度**:

| 层级 | 维度数 | 说明 |

| A 投资组合 | 3 | 仓位、风控、多样化 |

|------|------|------|

### 7.2 智能集成端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/integration/status` | GET | 整体状态 |
| `/integration/decision` | POST | 双脑决策 |
| `/integration/mirofish/run` | GET | 触发仿真 |
| `/integration/mirofish/report` | GET | 仿真报告 |
| `/integration/gstack/dispatch/{role}` | GET | 任务分发 |

### 7.3 调用示例

```bash
# 查看系统状态
curl http://localhost:8004/api/stats

# 查看市场行情
curl http://localhost:8004/api/market

# 触发MiroFish仿真
curl http://localhost:8004/integration/mirofish/run

# 双脑决策
curl -X POST http://localhost:8004/integration/decision \
  -H "Content-Type: application/json" \
  -d '{"symbol":"BTC/USDT","signal":"buy","confidence_left":0.7,"confidence_right":0.8}'
```

| 问题 | 解决方法 |
|------|---------|
| 页面打不开 | 重启后端: `bash scripts/start_server.sh` |
| API无响应 | 检查端口: `curl http://localhost:8004/api/stats` |
| 信号不更新 | 检查交易所连接 |
| 仿真失败 | 手动运行: `python3 scripts/mirofish_full_simulation_v2.py` |

## 9. 安全须知

### 9.1 API Key 保护
- API Key 只存储在服务器端
- 永不明文显示
- 定期轮换（90天）

### 9.2 操作审计
- 所有高风险操作记录到审计日志
- 审计日志路径: `/tmp/go2se_audit.log`

### 9.3 冻结机制
- 发现异常可激活冻结保护
- 冻结后禁止所有交易操作
- 使用 `/freeze` 命令激活
