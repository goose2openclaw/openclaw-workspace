## 🔴 问题一: E2-前端UI服务 (0.0分) — 假阴性

### 现象
```
test_frontend_ui() → FAIL (0.0分)
原因: 检查 localhost:5173 (Vite) → 无响应
实际: UI已嵌入Backend (localhost:8004)
```

### 根因
测试脚本检查 `localhost:5173` (Vite开发服务器)，但GO2SE的UI通过 `app.main:app` 后端服务提供，无需独立Vite服务器。

### 假阴性分析
- ✅ Backend的 `/` 根路径正常返回HTML
- ✅ 前端通过API正确加载数据
- **结论**: UI服务正常，测试设计有误

### 修复方案
修改测试脚本，将 `FRONTEND_URL` 改为实际端口:
```python
# mirofish_full_simulation_v2.py
FRONTEND_URL = "http://localhost:8004"  # 直接用后端端口
```

## 🔴 问题二: B1-打兔子工具 (40.8分) — 真实问题

### 现象
```
主流币回测数据 (共1728条):
  BTC/USDT: n=576, 收益=-1.24%, 胜率=31.0%
  ETH/USDT: n=576, 收益=-1.27%, 胜率=33.1%
  SOL/USDT: n=576, 收益=-1.69%, 胜率=28.4%
评分: 40.8/100 ⚠️ 警告
```

### 根因分析
| 指标 | 当前值 | 问题 |
|------|--------|------|
| 平均收益 | -1.40% | 负收益，策略整体亏损 |
| 平均胜率 | 30.9% | 低于50%，盈不抵亏 |
| 止损 | 5% | 可能过于宽松 |
| 止盈 | 8% | 可能过于保守 |

**策略参数问题**:
1. 止损5%过大 → 一次亏损吞噬多次盈利
2. 止盈8%过小 → 趋势行情抓不住
3. 缺少趋势过滤 → 在震荡市中反复被套

### 修复方案
```yaml
打兔子策略优化:
  止损: 5% → 2%    # 收紧止损
  止盈: 8% → 12%   # 放宽止盈，让利润奔跑
  新增: 趋势过滤    # 只在趋势确认时入场
  新增: RSI过滤     # RSI>70不追多，RSI<30不追空
```
