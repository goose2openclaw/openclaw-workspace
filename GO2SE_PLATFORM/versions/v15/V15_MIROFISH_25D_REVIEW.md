## 🔴 问题发现

### P1: Mi封顶1.20，但计算Mi已达上限

**代码**:
```python
mi = min(mi, 1.20)  # Mi=1.20封顶
```

**问题**: 
- 实际Mi计算公式 `(weighted/total_w + 0.5)` 理论最大值=2.0
- 封顶1.20意味着所有信号增强最多+20%
- 如果B1/B4/C1/D2都是88-90分的高分脑，Mi应该可以到1.30-1.40

**建议**: 
```python
mi = min(mi, 1.35)  # 提高到1.35，允许更大增强
```

---

### P2: B7_crowdsource权重2%但实际贡献仅1.2%

**问题**: 
- B7权重2%=0.02，在25维中最低
- 但crowdsource在7工具策略中是第7个工具
- 2%权重与其实际策略地位不匹配

**建议**: 提升到5%，与B5_hitchhiker(B5=4%)合并重新分配

---

### P3: A1_position权重8%但实际评分偏低(75分)

**问题**: 
- A1是投资组合的核心(8%权重最高)
- 但75分说明position sizing/管理评分不高
- Mi封顶1.20时A1的高权重无法充分发挥

**建议**: 
1. 提高position scoring标准到85分
2. 放开Mi上限让A1贡献更大

---

### P4: C4_sentiment 4%权重只有70分

**C4情绪模块** 当前评分70分（偏负面）：
- sentiment analysis在情绪极端时信号相反
- 当前v15无真实Twitter/新闻情绪输入
- C4是唯一一个只有4%却只有70分的维度

**建议**: 
- 若无法接入真实情绪API，删除C4或将其权重合并到C1_sonar(7%)
- C1_sonar已有趋势判断能力，C4补充价值低

---

### P5: E1_api 2%权重，基础设施权重过低

**问题**:
- E1-E6总共10%权重分了6个维度
- 每个只有1-2%，无法形成有效信号
- E1_api(2%)对高频交易决策质量影响被严重低估

**建议**: 合并为3个维度，每3-4%:
- E_tech = E1+E2+E3 (6%)  
- E_ops = E4+E5 (3%)
- E_perf = E6 (1%)
